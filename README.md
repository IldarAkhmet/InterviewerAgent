# InterviewerAgent — Multi-Agent Interview Coach

## Описание проекта

InterviewerAgent — это мультиагентная система для проведения технического интервью, имитирующая поведение реального интервьюера. Проект реализует **agentic-workflow**, в котором разные агенты выполняют специализированные роли, обмениваются скрытыми сигналами и принимают решения на основе контекста диалога, а не одного большого промпта.

Система ведёт диалог в формате «вопрос → ответ», адаптирует сложность вопросов, сохраняет контекст интервью и формирует структурированный финальный фидбэк по итогам сессии.

---

## Общая архитектура

Архитектура построена вокруг **оркестратора**, который управляет жизненным циклом интервью и координирует работу специализированных агентов.

Основные сущности:

- Оркестратор «Интервьюер»
- Создатель вопросов
- Обработчик ответов
- Агент финального фидбэка
- Память интервью

Каждая сущность отвечает только за свою зону ответственности, что соответствует принципу ролевой специализации.

---

## Сущности и их функции

### Оркестратор «Интервьюер»

Центральная управляющая сущность системы.

Функции:
- принимает входные данные о кандидате (позиция, грейд, опыт);
- инициализирует и координирует агентов;
- управляет порядком шагов интервью;
- прокидывает контекст между агентами;
- логгирует все сообщения и внутренние решения;
- сохраняет данные в память интервью.

---

### Создатель вопросов (Question Generator)

Отвечает за генерацию следующего вопроса интервью.

Функции:
- анализирует последние сообщения пользователя;
- не задаёт повторяющиеся вопросы;
- учитывает:
  - среднюю оценку последних ответов;
  - оценку за последний ответ;
  - выявленные пробелы в знаниях;
- адаптирует сложность:
  - при низкой средней оценке — упрощает вопросы;
  - при слабом последнем ответе — фокусируется на knowledge gaps;
- всегда генерирует **ровно один вопрос**;
- возвращает результат в структурированном JSON-формате.

---

### Обработчик ответов (Answer Evaluator)

Отвечает за оценку ответа кандидата на заданный вопрос.

Функции:
- использует историю последних N вопросов и ответов;
- оценивает ответ по нескольким критериям:
  - техническая корректность,
  - ясность,
  - глубина;
- выявляет пробелы в знаниях;
- определяет, хочет ли пользователь остановить интервью;
- корректно обрабатывает:
  - meta/job вопросы (без снижения оценки);
  - off-topic ответы;
- возвращает результат в строгом JSON-формате:
  - review,
  - score,
  - knowledge_gaps,
  - confidence_level,
  - stop_interview,
  - llm_answer.

---

### Агент финального фидбэка (Feedback Agent)

Агент пост-анализа интервью.

Функции:
- принимает полный лог интервью;
- анализирует совокупность ответов кандидата;
- формирует итоговый структурированный отчёт:
  - вердикт по уровню кандидата;
  - анализ hard skills;
  - анализ soft skills;
  - уверенность оценки.

---

### Память интервью (Interview Memory)

Отвечает за сохранение и предоставление контекста.

Функции:
- хранит историю диалога;
- сохраняет оценки, пробелы в знаниях и метрики;
- позволяет агентам учитывать сообщения, полученные ранее;
- предотвращает повторение вопросов и потерю контекста.

---

## Workflow интервью

1. Оркестратор принимает вводные данные кандидата.
2. Создатель вопросов формирует следующий вопрос.
3. Пользователь отвечает.
4. Обработчик ответов:
   - оценивает ответ,
   - обновляет память,
   - формирует скрытую рефлексию.
5. Оркестратор принимает решение:
   - продолжить интервью,
   - либо завершить и вызвать агент финального фидбэка.
6. При завершении формируется итоговый отчёт.

---

## Hidden Reflection и логирование

Система логгирует не только пользовательские сообщения, но и:
- внутренние оценки;
- решения агентов;
- сигналы между агентами.

Это позволяет:
- отследить, почему был задан конкретный вопрос;
- понять, как менялась сложность интервью;
- увидеть скрытый диалог агентов.

Для этого используется файл `interview_log.json`.

---

## Работа с meta/job вопросами

Если пользователь задаёт вопрос, связанный с:
- вакансией,
- ролью,
- процессом интервью,
- условиями работы,

то:
- оценка (`score`) не снижается;
- ответ формируется в поле `llm_answer`;
- интервью продолжается без искажения метрик.

## Текущее состояние репозитория

Репозиторий содержит:
- каркас проекта;
- зависимости (`requirements.txt`);
- конфигурацию окружения (`.env`);
- артефакты логирования (`interview_log.json`);
- задел под агентную архитектуру.

Проект сфокусирован на архитектуре, agentic-workflow и корректном разделении ролей между агентами.

---

## Итог

InterviewerAgent реализует мультиагентную архитектуру технического интервью с:
- ролевой специализацией,
- скрытой рефлексией,
- сохранением контекста,
- адаптацией сложности,
- структурированным финальным анализом.

Проект не является «умным чат-ботом», а представляет собой управляемую систему взаимодействующих агентов, приближенную к реальному процессу технического интервью.
